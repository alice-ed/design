When you build prototypes in code, for example, using the prototype kit, you should store them as repositories in the DfE GitHub instance.

Using GitHub to store your prototypes will ensure you're working towards meeting Service Standard 12 - Make new source code open.

Getting access to GitHub

If you don't have an existing GitHub account, sign up for a free account.

If you have an existing GitHub account, or, you've created a new one, you can request to be added to the DfE organisation GitHub area.

Request access to the DfE GitHub organisation

We have a DfE organisation instance in GitHub which means all DfE-related code can be stored securely in one place, to get access, submit a Service Now request

  1. go to Digital Tools Support in Slack
  2. select Digital Tools Requests from the pinned options (top of the channel)
  3. choose GitHub - Service Portal

Once you have access, you'll be able to view all public and private projects and code (also called repositories), and create new repositories.

Naming your repository

You should include the word prototype in your repository name to prevent people getting confused with production services.

Use the name of your service, in lowercase, separated by hyphens, followed by the word prototype.

Example reposititory name

apply-for-funding-prototype

Hosting your prototypes

Once you have your prototype code stored in GitHub, you can then get it up and running as a working website using Heroku.

You'll also be able to set it up so that any new changes get automatically deployed when you push changes to code, up to GitHub.
